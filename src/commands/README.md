# commands/

CLI entrypoints, wired up by `src/index.ts` (commander). Each command constructs its own deps (config → backend → embedder → local store) and closes them on exit.

- **`search.ts`** — `engram search <query> [--repo --branch --since --tier --limit --rerank --json]`. Formatted output by default; `--json` emits pure JSON on stdout (diagnostics go to stderr) so it's always `jq`-safe. `--tier raw|dream|wiki|synth|all` (CLI default `all` — the power tool sees everything; `both` kept as a deprecated alias) filters by memory tier; dream hits are tagged `[dream:<type>]`, wiki hits render as `[wiki:<kind>] <slug>` with a provenance line. `--rerank` (opt-in, needs `OPENAI_API_KEY`) runs the LLM reranker over the top-K hybrid pool and appends `rank=#N` to each reordered result; falls back to hybrid order without a key.
- **`ask.ts`** — `engram ask <question> [--repo --branch --since --tier(default synth) --k(default 12) --json]`. One synthesized, citation-backed answer over the top-`k` hybrid hits (the [ask layer](../ask/README.md)). Checks `OPENAI_API_KEY` **before** any DB work and exits 1 with an `engram search` pointer if absent — ask cannot degrade to plain retrieval. Prints the answer, then only the **cited** sources (`[n] [wiki:kind] slug · date · chunk id`); zero candidates → an `engram search --tier all` hint and exit 0. `--json` = `{answer, sources[], usage}` (all sources with a `cited` flag); the usage line goes to stderr so stdout stays clean. Logs a `recents` row of kind `'ask'` before the LLM call **and** exactly one `demand_log` row per ask after it (surface `'cli'`, outcome from `askOutcome` — `answered` / `not_covered` / `no_candidates`, or `error` on `AskError`), the demand-driven-synthesis signal for roadmap #6 — a failed/unanswerable ask is still demand. Accepts an optional injected `{backend, embedder, llm, local}` (`AskCommandDeps`) so tests drive it with fakes + a temp store; when injected the caller owns the lifecycle (nothing is closed).
- **`demand.ts`** — `engram demand [--days 30] [--json]`. The unmet-demand report from the `demand_log` sqlite table (the shopping list for future capture / targeted synthesis): a header line (`N events · S searches · A asks · U unmet (Q queries)`) plus a grouped table of unmet queries — `count`, last-asked age, a `coverage` indicator (`raw` when a `top_session_id` exists ⇒ uncompiled material to synthesize against; `—` ⇒ nothing captured yet), and the normalized query. Unmet = asks that were `not_covered`/`no_candidates` + searches with zero hits or `top_similarity < UNMET_THRESHOLD`. `--json` emits `{summary, unmet}`. Accepts an optional injected `LocalStore` test seam.
- **`dream.ts`** — `engram dream [--repo --since --limit --owner --dream-owner --dry-run --json]`. Synthesizes the [dream layer](../dream/README.md) over raw chunks. `--dry-run` prints the unit plan + token estimate with zero LLM calls/writes; a real run needs `OPENAI_API_KEY` (no fallback — a dream layer can't be synthesized without a chat model) and reports `synthesized/skipped/deferred/failed` counts + token totals. Idempotent via the fingerprint short-circuit.
- **`wiki.ts`** — `engram wiki ingest|lint|status|reindex [--repo --since --limit --owner --wiki-owner --dry-run --llm --json]`. Compiles the [wiki layer](../wiki/README.md) from dream chunks. `ingest --dry-run` = plan + token estimate, zero cost; a real run needs `OPENAI_API_KEY`, takes the shared advisory lock (`~/.engram/synthesis.lock`; a held lock exits 0 with `{skipped:'locked'}`), and reports created/updated/guarded/skipped counts. `lint` runs deterministic checks (`--llm` adds a contradiction pass); `status` reports pages by kind, links/orphans, pending units, git head; `reindex` rebuilds the pg `tier='wiki'` index from the files.
- **`context.ts`** — `engram context [--repo --branch --cwd --budget --owner --json]`. Emits a compact, repo-scoped markdown block (relevant [wiki](../context/README.md) pages + recent decisions/gotchas) for injection at session start. Repo/branch resolve from `--repo` (wins), else the git checkout at `--cwd`, else `process.cwd()`. Hard token `--budget` (default 1500, clamped 100–20000), deterministic ordering, no LLM/embeddings (all SQL). **Silent-empty**: text mode prints nothing when nothing's relevant; `--json` always emits one parseable object. Any failure (bad config, DB down, git error) → one stderr line, empty stdout, **exit 0** — a hook must never inject noise or break a session start.
- **`hooks.ts`** — `engram hooks print [--json]`. Prints the exact `~/.claude/settings.json` snippet wiring a `SessionStart` hook to `engram context --cwd "$CLAUDE_PROJECT_DIR"` (schema verified against the official Claude Code hooks docs), plus a how-to (global vs per-project, the `--budget` knob, matcher tuning). Print-only — never edits settings files.
- **`synthesisRun.ts`** — `synthesis-run` (hidden, headless): dream → **demand-targeted dream** → wiki over anything new since `last_synthesis_at − 24h` (overlap is free — fingerprints skip). One JSON line per phase to stdout (launchd → `~/.engram/synthesis.log`); the nightly agent + watcher-hook target. Phases in order: `dream`, `demand`, `wiki`, `done` (or `skipped`/`error`). The **`demand` phase line** = `{days, total, searches, asks, unmet, unmetQueries, targetedSessions, targetedSynthesized, targetedDreamChunks}` — the 30-day `demandSummary` plus what the targeted pass did. **Demand-targeted compilation** runs *between* the main dream pass and wiki (so this same run's wiki ingest picks up the fresh units): from `unmetDemand(30)` it takes up to `MAX_TARGETED_SESSIONS` (5) **distinct** `top_session_id`s (raw coverage that queries never found compiled) and re-dreams each with a `sessionId` filter and **no `since` window** (old backfilled sessions must compile). Fingerprints make an already-compiled session a free skip, so a second night over the same demand no-ops; the targeted list is deduped + capped regardless (`selectTargetedSessions`, exported + unit-tested). With an empty demand log the phase line is all-zeros and no targeted passes run — behavior is otherwise identical to before. Injectable seams (`SynthesisRunDeps`: `config`, `local`, `acquireLock`, `collaborators` factory bundling `{backend, embedder, dreamLLM, wikiLLM, wikiStore}`, `synthesize`, `ingest`, `log`) mirror `buildUiFetch(UiDeps)` so the orchestration is tested with fakes + a temp store, no OpenAI key or live pg; an injected `local` is left open (creator closes). The advisory lock path is unchanged.
- The best-effort advisory lock shared by `dream`, `wiki ingest`, and `synthesis-run` (stale after 30 min) — so the watcher hook, nightly agent, and manual runs can't interleave LLM synthesis — now lives in [`../synthesis/lock.ts`](../synthesis/README.md) alongside the synthesis queue it serializes.
- **`status.ts`** — config completeness, key/db presence, watch path, synthesis toggle, last ingest time, total chunk count, wiki page count.
- **`backfill.ts`** — recursively finds `.jsonl` under the watch path and runs the ingest pipeline over each; reports chunks embedded per trajectory and embedding-cache hit/miss totals. Idempotent — cursors and seen-hashes make re-runs cheap.
- **`watch.ts`** — `watch-internal` (hidden): long-running `SessionWatcher` process, meant to be supervised by launchd / `dev.sh`. Fails fast (exits non-zero to the log) if config is incomplete, so a supervised run never blocks on a prompt.
- **`service.ts`** — `engram service install|uninstall|status [--dry-run]` (macOS/launchd), refactored around a generic `AgentSpec`/`buildPlist`/`installAgent`. Two agents: `com.engram.watcher` (`RunAtLoad`+`KeepAlive`, `watch-internal`) always installed, and `com.engram.synthesis` (`RunAtLoad: false` + `StartCalendarInterval {Hour: synthesis.hour, Minute: 0}`, `synthesis-run`) installed **iff** `config.synthesis.enabled` — and **removed** if present while disabled (toggle-off + `install`). `--dry-run` prints the plists that would be installed without touching launchctl. `uninstall` removes both; `status` reports both (loaded state, plist, schedule, last log lines). `buildPlist`/`watcherSpec`/`synthesisSpec` are exported for the plist unit test — no `launchctl bootstrap` in tests. The agent operations are factored out for reuse by the ui-server settings pane: `agentStatus(spec)` returns `{label, loaded, state, pid, plistPresent, schedule}` as data (CLI `status` renders it), `installAgent`/`uninstallAgent` throw/return instead of `process.exit` (lethal inside the always-on server — the CLI wrapper catches and exits(1)), `restartAgent(label)` runs `launchctl kickstart -k` with the label validated against the two known constants, `servicesStatus()` bundles both agents (non-darwin → clean `supported:false` payload, not `process.exit`), and `reconcileSynthesisAgent()` (re)installs/uninstalls the synthesis agent to match `config.synthesis` — but only if the watcher plist exists. `serviceCommand` stays a thin CLI wrapper; CLI behavior is unchanged.
- **`ui.ts`** — `engram ui [--port 7777]`: the local search-and-ask web server (`Bun.serve`, loopback-only). The request handler is `buildUiFetch(deps)` (exported for route tests, no port bind); the full route list lives in [ui/README](../ui/README.md). Beyond search/wiki/config/services it serves **`POST /api/ask`** (builds an `OpenAIAskLLM` per request from `loadConfig()` — no key → `503 no_api_key` — runs `runAsk`, returns `{answer, sources, usage, model, tookMs}`, `AskError` → 502, other → 500) and **`GET /api/demand?days`** (unmet-demand report, `days` clamped 1–365). `/api/search` and `/api/ask` both write `demand_log` rows via the same `surface:'ui'` path the CLI uses (search: every settled query `>=3` chars incl. zero-hit; ask: one row with the derived `outcome` — inline for now, `// TODO(wave-9)` to adopt `askOutcome`). `Bun.serve` sets `idleTimeout: 240` so a 5–60s ask isn't severed by Bun's 10s default (verified on 1.3.14). The ask LLM is injected via `deps.buildAskLLM` for no-network route tests.
