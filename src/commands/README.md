# commands/

CLI entrypoints, wired up by `src/index.ts` (commander). Each command constructs its own deps (config → backend → embedder → local store) and closes them on exit.

- **`search.ts`** — `engram search <query> [--repo --branch --since --tier --limit --rerank --json]`. Formatted output by default; `--json` emits pure JSON on stdout (diagnostics go to stderr) so it's always `jq`-safe. `--tier raw|dream|wiki|synth|all` (CLI default `all` — the power tool sees everything; `both` kept as a deprecated alias) filters by memory tier; dream hits are tagged `[dream:<type>]`, wiki hits render as `[wiki:<kind>] <slug>` with a provenance line. `--rerank` (opt-in, needs `OPENAI_API_KEY`) runs the LLM reranker over the top-K hybrid pool and appends `rank=#N` to each reordered result; falls back to hybrid order without a key.
- **`ask.ts`** — `engram ask <question> [--repo --branch --since --tier(default synth) --k(default 12) --json]`. One synthesized, citation-backed answer over the top-`k` hybrid hits (the [ask layer](../ask/README.md)). Checks `OPENAI_API_KEY` **before** any DB work and exits 1 with an `engram search` pointer if absent — ask cannot degrade to plain retrieval. Prints the answer, then only the **cited** sources (`[n] [wiki:kind] slug · date · chunk id`); zero candidates → an `engram search --tier all` hint and exit 0. `--json` = `{answer, sources[], usage}` (all sources with a `cited` flag); the usage line goes to stderr so stdout stays clean. Logs a `recents` row of kind `'ask'` (the demand-driven-synthesis signal for roadmap #6) before the LLM call — a failed/unanswerable ask is still demand.
- **`dream.ts`** — `engram dream [--repo --since --limit --owner --dream-owner --dry-run --json]`. Synthesizes the [dream layer](../dream/README.md) over raw chunks. `--dry-run` prints the unit plan + token estimate with zero LLM calls/writes; a real run needs `OPENAI_API_KEY` (no fallback — a dream layer can't be synthesized without a chat model) and reports `synthesized/skipped/deferred/failed` counts + token totals. Idempotent via the fingerprint short-circuit.
- **`wiki.ts`** — `engram wiki ingest|lint|status|reindex [--repo --since --limit --owner --wiki-owner --dry-run --llm --json]`. Compiles the [wiki layer](../wiki/README.md) from dream chunks. `ingest --dry-run` = plan + token estimate, zero cost; a real run needs `OPENAI_API_KEY`, takes the shared advisory lock (`~/.engram/synthesis.lock`; a held lock exits 0 with `{skipped:'locked'}`), and reports created/updated/guarded/skipped counts. `lint` runs deterministic checks (`--llm` adds a contradiction pass); `status` reports pages by kind, links/orphans, pending units, git head; `reindex` rebuilds the pg `tier='wiki'` index from the files.
- **`context.ts`** — `engram context [--repo --branch --cwd --budget --owner --json]`. Emits a compact, repo-scoped markdown block (relevant [wiki](../context/README.md) pages + recent decisions/gotchas) for injection at session start. Repo/branch resolve from `--repo` (wins), else the git checkout at `--cwd`, else `process.cwd()`. Hard token `--budget` (default 1500, clamped 100–20000), deterministic ordering, no LLM/embeddings (all SQL). **Silent-empty**: text mode prints nothing when nothing's relevant; `--json` always emits one parseable object. Any failure (bad config, DB down, git error) → one stderr line, empty stdout, **exit 0** — a hook must never inject noise or break a session start.
- **`hooks.ts`** — `engram hooks print [--json]`. Prints the exact `~/.claude/settings.json` snippet wiring a `SessionStart` hook to `engram context --cwd "$CLAUDE_PROJECT_DIR"` (schema verified against the official Claude Code hooks docs), plus a how-to (global vs per-project, the `--budget` knob, matcher tuning). Print-only — never edits settings files.
- **`synthesisRun.ts`** — `synthesis-run` (hidden, headless): dream → wiki over anything new since `last_synthesis_at − 24h` (overlap is free — fingerprints skip). One JSON line per phase to stdout (launchd → `~/.engram/synthesis.log`); the nightly agent + watcher-hook target.
- **`synthesisLock.ts`** — best-effort advisory lock shared by `dream`, `wiki ingest`, and `synthesis-run` (stale after 30 min) so the watcher hook, nightly agent, and manual runs can't interleave LLM synthesis.
- **`status.ts`** — config completeness, key/db presence, watch path, synthesis toggle, last ingest time, total chunk count, wiki page count.
- **`backfill.ts`** — recursively finds `.jsonl` under the watch path and runs the ingest pipeline over each; reports chunks embedded per trajectory and embedding-cache hit/miss totals. Idempotent — cursors and seen-hashes make re-runs cheap.
- **`watch.ts`** — `watch-internal` (hidden): long-running `SessionWatcher` process, meant to be supervised by launchd / `dev.sh`. Fails fast (exits non-zero to the log) if config is incomplete, so a supervised run never blocks on a prompt.
- **`service.ts`** — `engram service install|uninstall|status [--dry-run]` (macOS/launchd), refactored around a generic `AgentSpec`/`buildPlist`/`installAgent`. Two agents: `com.engram.watcher` (`RunAtLoad`+`KeepAlive`, `watch-internal`) always installed, and `com.engram.synthesis` (`RunAtLoad: false` + `StartCalendarInterval {Hour: synthesis.hour, Minute: 0}`, `synthesis-run`) installed **iff** `config.synthesis.enabled` — and **removed** if present while disabled (toggle-off + `install`). `--dry-run` prints the plists that would be installed without touching launchctl. `uninstall` removes both; `status` reports both (loaded state, plist, schedule, last log lines). `buildPlist`/`watcherSpec`/`synthesisSpec` are exported for the plist unit test — no `launchctl bootstrap` in tests.
